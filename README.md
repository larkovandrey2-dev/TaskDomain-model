## Описание задачи
Реализовать модель задачи (`Task`) в рамках платформы обработки задач с корректной инкапсуляцией и валидацией состояния.
Ключевое условие: класс должен самостоятельно защищать свои инварианты (правильность данных и разрешенные переходы состояний) с помощью **пользовательских дескрипторов** и механизма **@property**, предоставляя чистый и безопасный публичный API.

## Архитектура классов

### 1. Перечисления и Исключения
*   **`TaskStatus(Enum)`**: Определяет возможные состояния задачи (`NEW`, `IN_PROGRESS`, `DONE`).
*   **`TaskDomainError(Exception)`**: Базовый класс для всех ошибок доменной модели.
*   **`ValidationError`** / **`StateError`**: Узкоспециализированные классы исключений для ошибок валидации атрибутов и нарушений графа состояний соответственно.

### 2. Пользовательские дескрипторы (Валидаторы)
*   **`IntegerRangeValidator`** *(Data Descriptor)*: Контролирует, что приоритет задачи является целым числом и не выходит за заданные границы (например, от 1 до 5).
*   **`StringValidator`** *(Data Descriptor)*: Гарантирует, что строковые атрибуты (описание задачи) передаются в верном типе и не являются пустыми.
*   **`StateValidator`** *(Data Descriptor)*: Блокирует недопустимые изменения статуса (например, попытку перевести задачу из `DONE` обратно в `NEW`).
*   **`TaskSummaryFormatter`** *(Non-data Descriptor)*: Генерирует текстовую сводку по задаче. Демонстрирует отсутствие метода `__set__`, что позволяет переопределить значение напрямую в словаре экземпляра объекта.

### 3. Доменная модель
*   **`Task`**: Класс задачи, который защищен от некорректных данных.
    *   Использует дескрипторы на уровне класса для `priority`, `description`, `state` и `summary`.
    *   Инкапсулирует `_id` и `_created_at` в момент инициализации, выдавая их наружу только через `@property` (только для чтения).
    *   Реализует бизнес-логику в вычисляемом свойстве `is_ready_for_execution`.

### Тестирование
Проект покрыт unit-тестами с использованием встроенного модуля `unittest`.
В тестах проверяется: успешная инициализация, блокировка перезаписи read-only свойств, выброс соответствующих исключений при передаче невалидных данных, соблюдение графа переходов машины состояний, а также поведение Non-data дескриптора.
